기간

2020.6.22 ~ 2020.6.26

프로젝트를 진행하면서 생각한 점을 남겨보려고 한다.

어디서부턴가 기술부채로 쌓여있던 자바스크립트들을 미니프로젝트를 진행하려고 보니 많은 부분에서 진행이 어려웠다. 확실히 이번 프로젝트들을 진행하면서 어려웠던 점은 알고리즘을 잘 세우지 못한다는 점, 그런 내 자신을 정확히 파악하지 못했던 점이다.

이번에 내가 맡은 부분은 리스트 부분인데, API에서 데이터들을 가지고 와서 특정 카테고리 별로 가져오고 해당 영상들의 정보들을 리스트에 출력한다. 또 그 리스트들이 슬라이드 되는 것을 맡았다. 이러한 부분에서 어려웠던 점과 프로젝트 기간동안 어떻게 해결했는지 적어본다.

어려웠던 점 / 해결방안

  1. 🎥 처음 제대로 접해 본 API

    오픈 API를 문서를 읽는 것부터가 난관이었다. 영어영어영어영어 블라블라블라.. 나는 내가 필요한 정보만 보고 무작정 코드를 작성했는데 하다 막히면 구글링부터 했다. 구글링으로 정답은 찾아냈지만 그 시간동안 너무 많은 시간이 소요됐고 이 정보가 확실한 정보인지도 의문이 갔다. 그래서 다른 조원한테 물어보니 일단 그 정보는 맞다고 했지만 이 정보가 API를 제공하는 문서에 있다고 했다. 😮???? 그렇다.. 나는 문서를 제대로 읽어보지 않아서 정답을 둘러둘러 찾았아 시간을 낭비해버렸다.

  2. 🤔 슬라이드 알고리즘이 이렇게 어려웠나?

    솔직히 이번 프로젝트를 통해서 API를 사용하여 화면에 출력하는 것 정도도 잘한거지만 매일 복붙으로 사용하는 슬라이드의 알고리즘을 짜볼까? 하는 욕심이 있었다. 기존엔 거의 복붙 수준으로 가져다 사용했지만 이번엔 구글링을 해서 나온 정보도 이해하면서 어떤 알고리즘으로 진행이 되는지 파악하면서 사용하려고 했다. 하지만 내가 원하는 슬라이드 기능은 20개의 리스트가 화면에 6개씩 보여지고 3번의 슬라이드 후 나머지 2개만큼만 슬라이드가 되는 기능을 원했는데 이런 기능이 있는 슬라이드를 구글링 하지 못했다. 구글링해서 나온 알고리즘과 내가 원하는 기능을 추가해서 해보려고 해도 결론적으로는 구현하지 못했다.

    그래서 결국 다른 조원에게 도움 요청을 했다. 금요일 마감인데 도저히 목요일까지 끝낼 수 없다고 생각했고 계속 붙잡는 것도 주요 기능 중 하나인데 도움을 청하는게 맞다고 생각했다. 그리고 나는 세부적인 CSS를 담당했다.

  3. ⌛ 스케쥴을 잘못 짰나.. 시간이 부족하다.

    분명 할 수 있을 것이라고 작성한 스케쥴대로 일이 진행되지 않았다. 내가 생각하지 못했던 부분에서 자꾸 오류가 나고 일정이 늦춰지고 그러다보니 분명 엄청 바쁘고 열심히 했는데 하루안에 끝내야할 리스트들이 다음날로 미뤄져 있었다. 하루 밀리고 또 하루 밀리고 겉잡을수 없이 미뤄졌고 결국 처음에 기획 했던 많은 기능들을 포기하고 정말 중요한 기능들만 구동되게 하였다.

    결국 내가 생각한 계획에서 2.5배의 시간을 잡는게 내가 생각치도 못한 오류에 대해 해결할 시간을 벌어주는것이 맞다고 느꼈다.

배운점 / 느낀점

프로젝트를 진행하면서 절대 혼자였다면 할 수 없었던 부분들이 많았는데 조원들을 통해서 협력을 통해 많은 부분을 해결할 수 있었다.

처음 기획을 할때 중간중간 매우 어려운 부분들이 있었다면 의사소통의 문제였다. 동일한 목적을 가진 말을 해도 과정을 다르게 말하니 의사소통이 매우 힘들었다. 동의를 얻는 것 조차 매우 힘들었는데 동의를 얻고 문제를 해결하는 과정은 말해 무엇할까.. 그럼에도 불구하고 서로가 서로를 이해하려고 노력을 매우 했던 것 같다. 처음 어느정도 이해하지 못했던 부분도 한발자국 물러나서 다시 생각해보고 이해하는 과정이 다들 나름대로 있었던 것 같다.

프로젝트가 처음 생각했던 기능보다의 결과물은 안나왔지만 그렇게 서로 이해하며 프로젝트를 잘 진행한 것이 더 뿌듯한 결과가 아닐까 생각해본다. 프로젝트야 이 기간이 끝나고 더 개발할 수 있지만 사람과 사람의 틀어진 사이를 다시 복구하려면 컴퓨터처럼 단순하지 않으니까 말이다.

ERROR Render HTML failed

Posted on 2020-04-19 Edited on 2020-06-28 In Error , HEXO

전 글인 safari table tag 100% error 블로그를 작성하고 가상서버로 테스트를 해봤는데 무한 로딩이 일어나면서 블로그가 열리지 않았다. 에러메세지를 확인해보니,

🚫 ERROR Render HTML failed: index.html

나는 index.html을 업로드한적도, 수정한적도 없었는데 에러가 나서 구글에 폭풍 검색을 했을때, _config.yml을 일부 수정해라, enable을 false로 넣어라 등 이것저것 방안이 많아 따라해보았다.

하지만 결과는.. 같은 에러메세지였다.

정말 문제 찾으려고 최선을 다했는데 구글에서도 원하는 답이 안나와 몇 시간동안 낑낑거렸다. 그래서 마음을 가다듬고 저번주에는 잘 업로드 되었지만 현재와 다른 점을 먼저 찾았다. 처음에는 올린 글의 문제인지도 못찾아서 헤맸는데 저번주와 다른 점이 업로드 한 그 글 밖에 없다는 생각에 큰 숲부터 보고 파고 들어갔다.

  1. 새로운 블로그 글 업로드 => 실패
  2. 저번주에 올린 블로그 내용 다 지우고 후 업로드 => 실패
  3. 저번주에 올린 블로그 제목 ‘safari’로 수정 후 업로드 => 성공

아, 블로그에 제목의 문제구나 싶어서 뭐가 문제인지 고민했다.

  1. 제목이 길어서 그런가?
  2. 제목에 특수문자가 들어가서 그런가?

제목을 짧게 바꾸었을때는 잘 업로드 되었다. 근데 ‘100%’ 만 넣었을때, 같은 에러메세지가 떴다!

드디어 찾았다. 에러의 원인을!!!😎

% 이 특수문자!! 마치 HTML에서 entities를 넣는 것처럼 예약어를 사용하는 듯 했다. 그래서 %의 entities를 찾아 %를 대신하여 &#37; 삽입하였더니 에러메세지가 뜨지 않고 해결되었다.

결론

헥소타이틀에 특수문자를 사용할 경우 entities를 사용하자.

🧚‍♀️

이 문제를 해결하기 위해 5시간 넘게 고생했지만 구글에서도 나오지 않던 답을 하나하나 파들어가며 해결 했던 것이 좋은 경험이 된 것 같다. 해결했을때의 쾌감이란..😌

safari table tag width 100% error

Posted on 2020-04-13 Edited on 2020-06-28 In Error , CSS

프로젝트를 진행하다 크로스브라우징 확인 중에 다른 브라우저와 달리 safari에서 table 의 width:100% 값이 부모요소의 padding 값을 뺀 영역만 차지했다.

순서대로: 사파리, 파이어폭스, 크롬

원인을 열심히 분석해보았지만 결론적으로는 safari에서 table이 다른 브라우저와 구현되는게 다르다는 것이었다. 뭐, 안되면 이 방법 저 방법 찾아서 해결 하면 됐지만 원인이 너무 찝찝하였다. 뭐.. 구현이 그렇게 된다는데 가서 따질 수 없으니 여러가지 해결방안을 찾아보았다.

  1. 자바스크립트로 제어

    조원들끼리 머리 맡대고 짰던 방안으로 조원 중 한명이 타닥하고 코드를 짰다. 자바스크립트로 브라우저가 사파리일 경우에만 width값을 넣어주는 방식.

    그렇지만 강사님께서 이 방법은 비추하셨다. 차라리 CSS에서 오류를 해결하기보단 회피하는것이 더 나은 방안이라고 말씀해주셨다.

  1. table , th , td 의 display: block ,inline-block 으로 변경

    display를block으로 변경시키는 방법으로 부모의 크기값을 잘 가져온다. 하지만 테이블 배치가 다 바뀌기 때문에 하위요소들 또한 display를 변경시켜야 한다. 결국 태그만 의미적으로 table이지만 배치는 다시해야하는 아이러니한 상황이..

  1. CSS hack으로 제어

    와.. 해보면서 소리질렀던 방법. CSS hack이란 일부 브라우저의 버그를 위해 그 브라우저에만 CSS를 적용하는 방법이다. 지금 사용한 CSS는 사파리 브라우저일 경우에만 CSS가 적용되는 방법이다.

Posted on 2020-04-05

CSS content 속성

나는 content속성을 주로 가상요소 선택자인 ::after, ::before에 문자열 또는 빈 값으로 넣어 스타일링을 했다. 하지만 content에 들어가는 값들은 문자열 이외에도 많다.

content 속성 값 리스트

  • string
    대체 텍스트를 넣을 수 있는 값으로 모든 문자와 숫자가 가능하며 “” 안에 값을 넣어주면 된다.

  • image
    이미지를 넣을 수 있는 값으로 url("이미지경로")로 값을 넣으며, 이미지의 설명을 /로 구분하여 대체 텍스트를 넣을 수 있다.

  • counter
    자동으로 번호를 매길 수 있는 변수로 CSS규칙에 따라 값이 증가 또는 감소한다.

  • attr()
    요소의 속성을 문자열로 변환하여 브라우저에 보여 준다.

🧚
counter, attr() 값을 불러오는 것이 매우 흥미로웠다.
